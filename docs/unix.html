Appendix C: Unix Version

Files

Files are an important part of any editor, therefore, Zedit tries to make file name entry as easy as possible. The Zedit file commands all support file name completion. If you are typing in a file name and hit <tab>, Zedit will try to complete the file name much like Command Completion. A <space> will display all possible matches.

Zedit also understands:
.The current directory.
..The previous directory.
~Your home directory.
~<loginname>The home directory for loginname.

Zedit limits a full path name to 128 characters.

For backups, Zedit appends a ~ to the filename. On SYS5 machines, it will place the ~ at the 14th character.

Shell Commands

Zedit supports piped shells and makes on Unix machines. This allows you to have a shell running at the same time as you are editing. It also allows a make and editing at the same time or makes without leaving the editor. It also allows you to perform Next Error commands while the make is still in progress.

Keys

Zedit does not support as many keys as emacs. Aside from the normal control keys and meta key, Zedit supports the following keys:

  • Function keys F1 to F12 inclusive
  • The arrow keys
  • The navigation keys: Page Up/Down, Home/End, Insert/Delete
  • The Ctrl arrow keys
  • Ctrl Home and Ctrl End

Note: For termcap and terminfo only keys that have entries in the database are supported. ANSI supports all keys

Hint: For termcap and terminfo it helps if all the special keys start with the ESCAPE character. Zedit can then optimize key lookup.
